# ci_linux_clang_libcxx Job Analysis

## Overview

The `ci_linux_clang_libcxx` job is a manually-triggered Jenkins pipeline for running a ROS 2 build on Linux using the `clang` compiler and the `libc++` standard library. It is dynamically generated by the `create_jenkins_job.py` script and is an instance of the generic `ci_job` template.

## Job Generation

The `create_jenkins_job.py` script generates the `ci_linux_clang_libcxx` job specifically for the `linux` OS. It's designed to test the compilation of ROS 2 with a non-default compiler and standard library to catch potential compatibility and thread-safety issues.

## Clang/libc++-Specific Configuration

The `ci_linux_clang_libcxx` job has several key differences from the standard `ci_linux` job:

*   **Job Name:** The job is explicitly named `ci_linux_clang_libcxx`.
*   **Compile with Clang:** The `compile_with_clang_default` parameter is set to `true`. This triggers the installation of `clang`, `libc++-dev`, and `libc++abi-dev` within the Docker container.
*   **Build Arguments:** The job uses a specific set of `build_args_default`:
    *   It forces the build of vendor packages (`-DFORCE_BUILD_VENDOR_PKG=ON`).
    *   It uses the `clang-libcxx` colcon mixin (`--mixin clang-libcxx`).
    *   It skips several GUI-related packages (`--packages-skip intra_process_demo qt_gui_cpp rqt_gui_cpp qt_gui_core rqt`).
*   **Test Arguments:** The job has a very limited set of tests it runs:
    *   It only tests the `rcutils` package (`--packages-select rcutils`).
    *   It does not re-run tests on failure (`--retest-until-pass 2` is removed).

## Execution Flow

The execution flow of the `ci_linux_clang_libcxx` job is similar to the standard `ci_linux` job, with the main difference being the build environment inside the Docker container.

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job uses the standard `linux_docker_resources/Dockerfile`.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile` and tagged as `ros2_batch_ci`. The `COMPILE_WITH_CLANG=true` build argument is passed to the `docker build` command, which triggers the installation of the necessary clang and libc++ packages.
    *   **Docker Container Run:** The job runs the newly built Docker image, passing the specific build and test arguments via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py`:** The logic inside the container is the same as other `ci` jobs, orchestrated by the `ros2_batch_job` Python module.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `ci_linux_clang_libcxx` job.
*   **Jenkins Job Template:**
    *   `job_templates/ci_job.xml.em`: The template for the `ci_job`.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
*   **Docker configuration:**
    *   `linux_docker_resources/Dockerfile`: The Dockerfile for Linux builds.

## Parameters

The `ci_linux_clang_libcxx` job accepts the same parameters as the `ci_launcher` and its other downstream jobs, but many of them are overridden with specific values to configure the clang/libc++ build.
