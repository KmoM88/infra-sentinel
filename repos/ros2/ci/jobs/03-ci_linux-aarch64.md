# ci_linux-aarch64 Job Analysis

## Overview

The `ci_linux-aarch64` job is a manually-triggered Jenkins pipeline for running a ROS 2 build on the `aarch64` (ARM64) architecture. It is dynamically generated by the `create_jenkins_job.py` script and is an instance of the generic `ci_job` template.

## Job Generation

The `create_jenkins_job.py` script generates the `ci_linux-aarch64` job based on the `linux-aarch64` entry in the `os_configs` dictionary. This configuration specifies `aarch64`-specific settings.

## aarch64-Specific Configuration

The `ci_linux-aarch64` job has the following key differences from the standard `ci_linux` job:

*   **Jenkins Node Label:** It is configured to run on a Jenkins node with the `linux_aarch64` label.
*   **RMW Implementation:** It adds `rmw_connextdds` to the default set of ignored RMW implementations.
*   **Test Arguments:** It modifies the default test arguments to exclude certain tests (e.g., `mimick`) that may not be suitable for the `aarch64` platform.
*   **Docker Build Argument:** It passes the build argument `PLATFORM=aarch64` to the `docker build` command.

## Execution Flow

The execution flow of the `ci_linux-aarch64` job is very similar to the `ci_linux` job:

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job uses the standard `linux_docker_resources/Dockerfile`. The `PLATFORM` build argument is used inside the Dockerfile to conditionally install platform-specific dependencies.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile` and tagged as `ros2_batch_ci_aarch64`.
    *   **Docker Container Run:** The job runs the newly built Docker image, passing the build arguments via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py` and `ros2_batch_job`:** The logic inside the container is the same as other `ci` jobs, orchestrated by the `ros2_batch_job` Python module. This script handles repository fetching, building, and testing using `vcstool` and `colcon`.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `ci_linux-aarch64` job.
*   **Jenkins Job Template:**
    *   `job_templates/ci_job.xml.em`: The template for the `ci_job`.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
    *   `ros2_batch_job/`: The Python module containing the core build and test logic.
*   **Docker configuration:**
    *   `linux_docker_resources/Dockerfile`: The Dockerfile for Linux builds.

## Parameters

The `ci_linux-aarch64` job accepts the same parameters as the `ci_launcher` and its other downstream jobs. For a full list of parameters, please refer to the documentation for the `ci_launcher` job.
