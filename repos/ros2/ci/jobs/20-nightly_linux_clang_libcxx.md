# nightly_linux_clang_libcxx Job Analysis

## Overview

The `nightly_linux_clang_libcxx` job is a nightly-triggered Jenkins pipeline for building and testing ROS 2 on Linux using the `clang` compiler and the `libc++` standard library. It is dynamically generated by the `create_jenkins_job.py` script and uses the `ci_job.xml.em` template. This job is the automated, nightly version of the manually-triggered `ci_linux_clang_libcxx` job.

## Job Generation

The `create_jenkins_job.py` script generates the `nightly_linux_clang_libcxx` job for the `linux` OS. It's designed to ensure that a subset of ROS 2 packages can be compiled with `clang` and `libc++` on a nightly basis.

## Clang/libc++-Specific Configuration

The `nightly_linux_clang_libcxx` job has the following key configuration settings:

*   **Job Name:** The job is explicitly named `nightly_linux_clang_libcxx`.
*   **Template:** It uses the `ci_job.xml.em` template.
*   **Jenkins Node Label:** It is configured to run on a Jenkins node with the `linux` label.
*   **Build Type:** The `cmake_build_type` is set to `Debug`.
*   **Trigger:** The job is triggered by a timer (`time_trigger_spec`) set to run daily at 4:00 AM (`0 4 * * *`).
*   **Notifications:** Email notifications for build results are sent to `ros2-buildfarm@googlegroups.com` and `ros-contributions@amazon.com`.
*   **Compile with Clang:** The `compile_with_clang_default` parameter is set to `true`. This triggers the installation of `clang`, `libc++-dev`, and `libc++abi-dev` within the Docker container.
*   **Build Arguments:** The build arguments are heavily modified to enable clang and libc++:
    *   `--cmake-args -DFORCE_BUILD_VENDOR_PKG=ON`: This flag forces the build of vendor packages, which is necessary when using a different compiler.
    *   `--mixin clang-libcxx`: This `colcon` mixin provides the necessary compiler and linker flags to use clang and libc++.
    *   `--packages-skip intra_process_demo qt_gui_cpp rqt_gui_cpp qt_gui_core rqt`: This skips several GUI-related packages.
*   **Test Arguments:** The test arguments are also modified to limit the scope of the tests:
    *   `--packages-select rcutils`: This limits the tests to the `rcutils` package.
*   **Docker Image:** The Docker image is tagged as `ros2_batch_ci`.

## Execution Flow

The execution flow of the `nightly_linux_clang_libcxx` job is similar to the `ci_linux_clang_libcxx` job.

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job uses the standard `linux_docker_resources/Dockerfile`.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile` and tagged as `ros2_batch_ci`. The `COMPILE_WITH_CLANG=true` build argument is passed to the `docker build` command.
    *   **Docker Container Run:** The job runs the newly built Docker image, passing the specific build and test arguments via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py`:** The `ros2_batch_job` Python module within the container handles the build and test process.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `nightly_linux_clang_libcxx` job.
*   **Jenkins Job Template:**
    *   `job_templates/ci_job.xml.em`: The template for the CI job.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
*   **Docker configuration:**
    *   `linux_docker_resources/Dockerfile`: The Dockerfile for Linux builds.

## Parameters

The `nightly_linux_clang_libcxx` job accepts the same parameters as the `ci_linux_clang_libcxx` job, but it is typically run with the default values since it is triggered automatically.
