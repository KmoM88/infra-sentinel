# ci_packaging_windows Job Analysis

## Overview

The `ci_packaging_windows` job is a manually-triggered Jenkins pipeline for building and packaging ROS 2 on Windows. It is dynamically generated by the `create_jenkins_job.py` script and uses the `packaging_job.xml.em` template.

## Job Generation

The `create_jenkins_job.py` script generates the `ci_packaging_windows` job for the `windows` OS. It's designed to create Chocolatey packages for the ROS 2 packages in the workspace.

## Windows Packaging-Specific Configuration

The `ci_packaging_windows` job has several key differences from the standard `ci_windows` job:

*   **Job Name:** The job is explicitly named `ci_packaging_windows`.
*   **Template:** It uses the `packaging_job.xml.em` template.
*   **Build Type:** The `cmake_build_type` is set to `RelWithDebInfo`.
*   **Packaging Flag:** The `--packaging` argument is passed to the `run_ros2_batch.py` script, which triggers the packaging logic.
*   **Artifact Archiving:** The job is configured to archive the generated Chocolatey packages (`ws/ros2-package-*.*`).
*   **Docker Image:** The Docker image is tagged as `ros2_windows_ci_<ROS_DISTRO>`.

## Execution Flow

The execution flow of the `ci_packaging_windows` job is similar to the `ci_windows` job, with the addition of the packaging step.

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job uses the `windows_docker_resources/Dockerfile`.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile` and tagged as `ros2_windows_ci_<ROS_DISTRO>`.
    *   **Docker Container Run:** The job runs the newly built Docker image, passing the specific build and test arguments, including the `--packaging` flag, via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py`:** The `ros2_batch_job` Python module within the container handles the build process. When the `--packaging` flag is present, it performs the packaging steps after the build and tests have run, generating Chocolatey packages.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `ci_packaging_windows` job.
*   **Jenkins Job Template:**
    *   `job_templates/packaging_job.xml.em`: The template for the packaging job.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
*   **Docker configuration:**
    *   `windows_docker_resources/Dockerfile`: The Dockerfile for Windows builds.

## Parameters

The `ci_packaging_windows` job accepts the same parameters as the `ci_launcher` and its other downstream jobs, but some of them are overridden with specific values to configure the packaging build.
