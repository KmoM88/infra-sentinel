# packaging_linux-aarch64 Job Analysis

## Overview

The `packaging_linux-aarch64` job is a nightly-triggered Jenkins pipeline for building and packaging ROS 2 on the `aarch64` (ARM64) architecture. It is dynamically generated by the `create_jenkins_job.py` script and uses the `packaging_job.xml.em` template. This job is the automated, nightly version of the manually-triggered `ci_packaging_linux-aarch64` job.

## Job Generation

The `create_jenkins_job.py` script generates the `packaging_linux-aarch64` job for the `linux-aarch64` OS. It's designed to create Debian packages for the ROS 2 packages in the workspace on a nightly basis, specifically for the `aarch64` architecture.

## aarch64 Packaging-Specific Configuration

The `packaging_linux-aarch64` job has the following key configuration settings:

*   **Job Name:** The job is explicitly named `packaging_linux-aarch64`.
*   **Template:** It uses the `packaging_job.xml.em` template.
*   **Jenkins Node Label:** It is configured to run on a Jenkins node with the `linux_aarch64` label.
*   **Build Type:** The `cmake_build_type` is set to `RelWithDebInfo`.
*   **Trigger:** The job is triggered by a timer (`time_trigger_spec`) set to run daily at 4:00 AM (`0 4 * * *`).
*   **Notifications:** Email notifications for build results are sent to `ros2-buildfarm@googlegroups.com`.
*   **RMW Implementation:** It adds `rmw_connextdds` to the default set of ignored RMW implementations.
*   **Docker Build Argument:** It passes the build argument `PLATFORM=aarch64` to the `docker build` command.
*   **Docker Image:** The Docker image is tagged as `ros2_packaging_aarch64`.
*   **Packaging Flag:** The `--packaging` argument is passed to the `run_ros2_batch.py` script, which triggers the packaging logic.
*   **Artifact Archiving:** The job is configured to archive the generated Debian packages (`ws/ros2-package-*.*`).

## Execution Flow

The execution flow of the `packaging_linux-aarch64` job is identical to the `ci_packaging_linux-aarch64` job.

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job uses the standard `linux_docker_resources/Dockerfile`.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile` and tagged as `ros2_packaging_aarch64`. The `PLATFORM=aarch64` build argument is passed to the `docker build` command.
    *   **Docker Container Run:** The job runs the newly built Docker image, passing the specific build and test arguments, including the `--packaging` flag, via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py`:** The `ros2_batch_job` Python module within the container handles the build process. When the `--packaging` flag is present, it generates Debian packages for all the ROS 2 packages in the workspace.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `packaging_linux-aarch64` job.
*   **Jenkins Job Template:**
    *   `job_templates/packaging_job.xml.em`: The template for the packaging job.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
*   **Docker configuration:**
    *   `linux_docker_resources/Dockerfile`: The Dockerfile for Linux builds.

## Parameters

The `packaging_linux-aarch64` job accepts the same parameters as the `ci_packaging_linux-aarch64` job, but it is typically run with the default values since it is triggered automatically.
