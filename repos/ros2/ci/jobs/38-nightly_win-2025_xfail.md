# nightly_win-2025_xfail Job Analysis

## Overview

The `nightly_win-2025_xfail` job is a nightly-triggered Jenkins pipeline for building and testing ROS 2 on Windows Server 2025, with a specific focus on running tests that are expected to fail (`xfail`). It is dynamically generated by the `create_jenkins_job.py` script and uses the `ci_job.xml.em` template.

## Job Generation

The `create_jenkins_job.py` script generates the `nightly_win-2025_xfail` job for the `windows-2025` OS. It's designed to monitor the status of known failing tests on a nightly basis.

## XFail Testing-Specific Configuration

The `nightly_win-2025_xfail` job has the following key configuration settings:

*   **Job Name:** The job is explicitly named `nightly_win-2025_xfail`.
*   **Template:** It uses the `ci_job.xml.em` template.
*   **Jenkins Node Label:** It is configured to run on a Jenkins node with the `windows-2025-container` label.
*   **Build Type:** The `cmake_build_type` is set to `None`.
*   **Trigger:** The job is triggered by a timer (`time_trigger_spec`) set to run daily at 4:00 AM (`0 4 * * *`).
*   **Notifications:** Email notifications for build results are sent to `ros2-buildfarm@googlegroups.com`.
*   **Test Arguments:** The test arguments are significantly modified to only run `xfail` tests:
    *   `--ctest-args -L xfail`: This flag tells `ctest` to only run tests with the `xfail` label.
    *   `--pytest-args -m xfail --runxfail`: This tells `pytest` to only run tests marked with `xfail` and to run them even though they are expected to fail.
*   **Docker Image:** The Docker image is tagged as `ros2_windows_ci_<ROS_DISTRO>`.

## Execution Flow

The execution flow of the `nightly_win-2025_xfail` job is similar to the `ci_windows-2025` job, with the main difference being the set of tests that are executed.

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job uses the `windows_docker_resources/Dockerfile`.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile` and tagged as `ros2_windows_ci_<ROS_DISTRO>`.
    *   **Docker Container Run:** The job runs the newly built Docker image, passing the specific build and test arguments via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py`:** The `ros2_batch_job` Python module within the container handles the build and test process. The specific `ctest` and `pytest` arguments are passed to `colcon test`.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `nightly_win-2025_xfail` job.
*   **Jenkins Job Template:**
    *   `job_templates/ci_job.xml.em`: The template for the CI job.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
*   **Docker configuration:**
    *   `windows_docker_resources/Dockerfile`: The Dockerfile for Windows builds.

## Parameters

The `nightly_win-2025_xfail` job accepts the same parameters as the `ci_windows-2025` job, but it is typically run with the default values since it is triggered automatically.
