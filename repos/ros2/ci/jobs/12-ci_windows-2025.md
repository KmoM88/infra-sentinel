# ci_windows-2025 Job Analysis

## Overview

The `ci_windows-2025` job is a manually-triggered Jenkins pipeline for running a ROS 2 build on the Windows Server 2025 platform. It is dynamically generated by the `create_jenkins_job.py` script and is an instance of the generic `ci_job` template, similar to the `ci_windows` job.

## Job Generation

The `create_jenkins_job.py` script generates the `ci_windows-2025` job based on the `windows-2025` entry in the `os_configs` dictionary. This configuration specifies settings for the Windows Server 2025 environment.

## Windows Server 2025-Specific Configuration

The `ci_windows-2025` job is very similar to the `ci_windows` job, with the primary difference being the Jenkins node on which it runs:

*   **Jenkins Node Label:** It is configured to run on a Jenkins node with the `windows-2025-container` label. This ensures that the job is executed on a machine with the correct operating system.
*   **Shell Type:** The job's "Execute shell" step uses a `BatchFile` script.
*   **Default Settings:** The `use_isolated_default` parameter is set to `false`.
*   **Dockerfile:** It uses the same dedicated Dockerfile as the `ci_windows` job, located at `windows_docker_resources/Dockerfile`.
*   **Docker Image Tag:** The generated Docker image is tagged as `ros2_windows_ci_<ROS_DISTRO>`.
*   **Environment Management:** The build environment inside the Docker container is managed by `pixi`.

## Execution Flow

The execution flow of the `ci_windows-2025` job is identical to that of the `ci_windows` job:

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job selects the `windows_docker_resources/Dockerfile`. The `WINDOWS_RELEASE_VERSION` build argument in the Dockerfile is used to pull the correct Windows Server base image.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile` and tagged as `ros2_windows_ci_<ROS_DISTRO>`.
    *   **Docker Container Run:** The job runs the newly built Docker image with process isolation (`--isolation=process`). It passes the build arguments via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py` and `pixi`:** Inside the container, the main build logic is orchestrated by `pixi`, which runs the `run_ros2_batch.py` script. The `ros2_batch_job` Python module then handles repository fetching, building, and testing using `vcstool` and `colcon`.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `ci_windows-2025` job.
*   **Jenkins Job Template:**
    *   `job_templates/ci_job.xml.em`: The template for the `ci_job`.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
    *   `ros2_batch_job/`: The Python module containing the core build and test logic.
*   **Docker configuration:**
    *   `windows_docker_resources/Dockerfile`: The Dockerfile for Windows builds.
*   **Environment Configuration:**
    *   `pixi.toml`: A file (fetched from a URL) that defines the build environment and dependencies for `pixi`.

## Parameters

The `ci_windows-2025` job accepts the same parameters as the `ci_launcher` and its other downstream jobs. For a full list of parameters, please refer to the documentation for the `ci_launcher` job.
