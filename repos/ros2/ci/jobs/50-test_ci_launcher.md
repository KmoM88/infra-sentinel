# test_ci_launcher Job Analysis

## Overview

The `test_ci_launcher` job is a manually-triggered Jenkins pipeline that serves as an experimental version of the main `ci_launcher` job. It is dynamically generated by the `create_jenkins_job.py` script and uses the `ci_launcher_job.xml.em` template. This job is intended for developers to test changes to the launcher configuration or the downstream jobs it triggers without affecting the production CI pipeline.

## Job Generation

The `create_jenkins_job.py` script generates the `test_ci_launcher` job. It is designed as a sandbox for testing modifications to the entire CI launch process.

## Test Launcher-Specific Configuration

The `test_ci_launcher` job has the following key configuration settings:

*   **Job Name:** The job is explicitly named `test_ci_launcher`.
*   **Template:** It uses the `ci_launcher_job.xml.em` template.
*   **Jenkins Node Label:** It is configured to run on a Jenkins node with the `built-in || master` label.
*   **Trigger:** This is a manually triggered job and has no time-based trigger.
*   **Notifications:** Email notifications are not configured for this job.
*   **Downstream Jobs:** This job is configured to trigger the `test_ci_*` jobs for each supported operating system (`test_ci_linux`, `test_ci_linux-aarch64`, `test_ci_linux-rhel`, and `test_ci_windows`).

## Execution Flow

The execution flow of the `test_ci_launcher` job is identical to the `ci_launcher` job, but it triggers the `test_ci_*` jobs instead of the `ci_*` jobs.

1.  **Groovy Script:** The first step in the `test_ci_launcher` job is to run a Groovy script that predicts the build numbers of the downstream `test_ci_*` builds that are about to be triggered.

2.  **Downstream Job Triggering:** The `test_ci_launcher` job uses the "Parameterized Trigger Plugin" to trigger a `test_ci_*` job for each supported platform. It passes all of its own parameters to these downstream jobs.

## Docker Image

The `test_ci_launcher` job itself does **not** spin up a Docker image. It runs directly on a Jenkins node. However, the downstream `test_ci_*` jobs that it triggers **do** spin up new Docker images as part of their execution.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `test_ci_launcher` job.
*   **Jenkins Job Template:**
    *   `job_templates/ci_launcher_job.xml.em`: The template for the launcher job.

## Parameters

The `test_ci_launcher` job accepts the same parameters as the `ci_launcher` job, allowing developers to test a wide range of build and test configurations across all `test_ci_*` jobs.
