# test_linux_coverage Job Analysis

## Overview

The `test_linux_coverage` job is a manually-triggered Jenkins pipeline for building and testing ROS 2 on Linux and generating code coverage reports. It is dynamically generated by the `create_jenkins_job.py` script and uses the `ci_job.xml.em` template. This job is intended for developers to experiment with changes to the coverage configuration without affecting the main `ci_linux_coverage` job.

## Job Generation

The `create_jenkins_job.py` script generates the `test_linux_coverage` job for the `linux` OS. It's designed as a sandbox for testing modifications to the coverage analysis process.

## Test Coverage-Specific Configuration

The `test_linux_coverage` job has the following key configuration settings:

*   **Job Name:** The job is explicitly named `test_linux_coverage`.
*   **Template:** It uses the `ci_job.xml.em` template.
*   **Jenkins Node Label:** It is configured to run on a Jenkins node with the `linux` label.
*   **Build Type:** The `cmake_build_type` is set to `Debug`.
*   **Trigger:** This is a manually triggered job and has no time-based trigger.
*   **Notifications:** Email notifications are not configured for this job.
*   **Enable Coverage:** The `enable_coverage_default` parameter is set to `true`.
*   **Build and Test Arguments:** The job uses a specific set of build and test arguments to focus on a curated list of packages (`quality_level_pkgs` and `testing_pkgs_for_quality_level`).
*   **Docker Image:** The Docker image is tagged as `ros2_batch_ci`.

## Execution Flow

The execution flow of the `test_linux_coverage` job is identical to the `ci_linux_coverage` job.

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job uses the standard `linux_docker_resources/Dockerfile`.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile` and tagged as `ros2_batch_ci`.
    *   **Docker Container Run:** The job runs the newly built Docker image, passing the specific build and test arguments via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py`:** The `ros2_batch_job` Python module within the container handles the build process and generates coverage reports.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `test_linux_coverage` job.
*   **Jenkins Job Template:**
    *   `job_templates/ci_job.xml.em`: The template for the CI job.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
*   **Docker configuration:**
    *   `linux_docker_resources/Dockerfile`: The Dockerfile for Linux builds.

## Parameters

The `test_linux_coverage` job accepts the same parameters as the `ci_linux_coverage` job, allowing developers to test a wide range of coverage configurations.
