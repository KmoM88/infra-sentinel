# ci_packaging_linux-aarch64 Job Analysis

## Overview

The `ci_packaging_linux-aarch64` job is a manually-triggered Jenkins pipeline for building and packaging ROS 2 on the `aarch64` (ARM64) architecture. It is dynamically generated by the `create_jenkins_job.py` script and uses the `packaging_job.xml.em` template.

## Job Generation

The `create_jenkins_job.py` script generates the `ci_packaging_linux-aarch64` job for the `linux-aarch64` OS. It's designed to create Debian packages for the ROS 2 packages in the workspace, specifically for the `aarch64` architecture.

## aarch64 Packaging-Specific Configuration

The `ci_packaging_linux-aarch64` job has several key differences from the standard `ci_packaging_linux` job:

*   **Job Name:** The job is explicitly named `ci_packaging_linux-aarch64`.
*   **Jenkins Node Label:** It is configured to run on a Jenkins node with the `linux_aarch64` label.
*   **RMW Implementation:** It adds `rmw_connextdds` to the default set of ignored RMW implementations.
*   **Docker Build Argument:** It passes the build argument `PLATFORM=aarch64` to the `docker build` command.
*   **Docker Image:** The Docker image is tagged as `ros2_packaging_aarch64`.
*   **Build Type:** The `cmake_build_type` is set to `RelWithDebInfo`.
*   **Connext:** `use_connext_debs_default` is set to `true`, which means it will try to install Connext from Debian packages (though this is unlikely to happen on `aarch64`).
*   **Packaging Flag:** The `--packaging` argument is passed to the `run_ros2_batch.py` script, which triggers the packaging logic.
*   **Artifact Archiving:** The job is configured to archive the generated Debian packages (`ws/ros2-package-*.*`).

## Execution Flow

The execution flow of the `ci_packaging_linux-aarch64` job is similar to the `ci_packaging_linux` job, with the main difference being the target architecture.

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job uses the standard `linux_docker_resources/Dockerfile`.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile` and tagged as `ros2_packaging_aarch64`. The `PLATFORM=aarch64` build argument is passed to the `docker build` command.
    *   **Docker Container Run:** The job runs the newly built Docker image, passing the specific build and test arguments, including the `--packaging` flag, via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py`:** The `ros2_batch_job` Python module within the container handles the build process. When the `--packaging` flag is present, it performs the packaging steps after the build and tests have run.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `ci_packaging_linux-aarch64` job.
*   **Jenkins Job Template:**
    *   `job_templates/packaging_job.xml.em`: The template for the packaging job.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
*   **Docker configuration:**
    *   `linux_docker_resources/Dockerfile`: The Dockerfile for Linux builds.

## Parameters

The `ci_packaging_linux-aarch64` job accepts the same parameters as the `ci_launcher` and its other downstream jobs, but some of them are overridden with specific values to configure the packaging build.
