# test_packaging_linux-aarch64 Job Analysis

## Overview

The `test_packaging_linux-aarch64` job is a manually-triggered Jenkins pipeline for building and packaging ROS 2 on the `aarch64` (ARM64) architecture. It is dynamically generated by the `create_jenkins_job.py` script and uses the `packaging_job.xml.em` template. This job is intended for developers to experiment with changes to the packaging configuration for the `aarch64` architecture without affecting the main `packaging_linux-aarch64` job.

## Job Generation

The `create_jenkins_job.py` script generates the `test_packaging_linux-aarch64` job for the `linux-aarch64` OS. It's designed as a sandbox for testing modifications to the packaging process for `aarch64`.

## Test Packaging-Specific Configuration

The `test_packaging_linux-aarch64` job has the following key configuration settings:

*   **Job Name:** The job is explicitly named `test_packaging_linux-aarch64`.
*   **Template:** It uses the `packaging_job.xml.em` template.
*   **Jenkins Node Label:** It is configured to run on a Jenkins node with the `linux_aarch64` label.
*   **Build Type:** The `cmake_build_type` is set to `RelWithDebInfo`.
*   **Trigger:** This is a manually triggered job and has no time-based trigger.
*   **Notifications:** Email notifications are not configured for this job.
*   **Packaging Flag:** The `--packaging` argument is passed to the `run_ros2_batch.py` script, which triggers the packaging logic.
*   **Docker Image:** The Docker image is tagged as `ros2_packaging_aarch64`.
*   **Docker Build Argument:** It passes the build argument `PLATFORM=aarch64` to the `docker build` command.
*   **RMW Implementation:** It adds `rmw_connextdds` to the default set of ignored RMW implementations.
*   **Artifact Archiving:** The job is configured to archive the generated Debian packages (`ws/ros2-package-*.*`).

## Execution Flow

The execution flow of the `test_packaging_linux-aarch64` job is identical to the `packaging_linux-aarch64` job.

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job uses the standard `linux_docker_resources/Dockerfile`.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile` and tagged as `ros2_packaging_aarch64`.
    *   **Docker Container Run:** The job runs the newly built Docker image, passing the build and test arguments, including the `--packaging` flag, via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py`:** The `ros2_batch_job` Python module within the container handles the build and test process, generating Debian packages.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `test_packaging_linux-aarch64` job.
*   **Jenkins Job Template:**
    *   `job_templates/packaging_job.xml.em`: The template for the packaging job.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
*   **Docker configuration:**
    *   `linux_docker_resources/Dockerfile`: The Dockerfile for Linux builds.

## Parameters

The `test_packaging_linux-aarch64` job accepts the same parameters as the `packaging_linux-aarch64` job, allowing developers to test a wide range of packaging configurations.
