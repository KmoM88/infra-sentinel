# ci_packaging_linux-rhel Job Analysis

## Overview

The `ci_packaging_linux-rhel` job is a manually-triggered Jenkins pipeline for building and packaging ROS 2 on a Red Hat Enterprise Linux (RHEL) like environment. It is dynamically generated by the `create_jenkins_job.py` script and uses the `packaging_job.xml.em` template.

## Job Generation

The `create_jenkins_job.py` script generates the `ci_packaging_linux-rhel` job for the `linux-rhel` OS. It's designed to create RPM packages for the ROS 2 packages in the workspace.

## RHEL Packaging-Specific Configuration

The `ci_packaging_linux-rhel` job has several key differences from the standard `ci_packaging_linux` job:

*   **Job Name:** The job is explicitly named `ci_packaging_linux-rhel`.
*   **Jenkins Node Label:** It is configured to run on a Jenkins node with the `linux` label.
*   **Build Type:** The `cmake_build_type` is set to `RelWithDebInfo`.
*   **Connext:** `use_connext_debs_default` is set to `false`.
*   **Build Arguments:** The job passes an additional CMake argument: `-DPython3_EXECUTABLE=/usr/bin/python3`.
*   **Packaging Flag:** The `--packaging` argument is passed to the `run_ros2_batch.py` script, which triggers the packaging logic.
*   **Docker Image:** The Docker image is built from `linux_docker_resources/Dockerfile-RHEL` and tagged as `ros2_packaging_rhel`.
*   **Artifact Archiving:** The job is configured to archive the generated RPM packages (`ws/ros2-package-*.*`).

## Execution Flow

The execution flow of the `ci_packaging_linux-rhel` job is similar to the `ci_packaging_linux` job, with the main difference being the target distribution.

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job uses the `linux_docker_resources/Dockerfile-RHEL`.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile-RHEL` and tagged as `ros2_packaging_rhel`.
    *   **Docker Container Run:** The job runs the newly built Docker image, passing the specific build and test arguments, including the `--packaging` flag, via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py`:** The `ros2_batch_job` Python module within the container handles the build process. When the `--packaging` flag is present, it performs the packaging steps after the build and tests have run, generating RPM packages.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `ci_packaging_linux-rhel` job.
*   **Jenkins Job Template:**
    *   `job_templates/packaging_job.xml.em`: The template for the packaging job.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
*   **Docker configuration:**
    *   `linux_docker_resources/Dockerfile-RHEL`: The Dockerfile for RHEL builds.

## Parameters

The `ci_packaging_linux-rhel` job accepts the same parameters as the `ci_launcher` and its other downstream jobs, but some of them are overridden with specific values to configure the packaging build. The `CI_EL_RELEASE` parameter is used to select the specific RHEL/AlmaLinux version for the build.
