# ci_packaging_linux Job Analysis

## Overview

The `ci_packaging_linux` job is a manually-triggered Jenkins pipeline for building and packaging ROS 2 on Linux. It is dynamically generated by the `create_jenkins_job.py` script and uses a specific `packaging_job.xml.em` template.

## Job Generation

The `create_jenkins_job.py` script generates the `ci_packaging_linux` job for the `linux` OS. It's designed to create Debian packages for the ROS 2 packages in the workspace.

## Packaging-Specific Configuration

The `ci_packaging_linux` job has several key differences from the standard `ci_linux` job:

*   **Job Name:** The job is explicitly named `ci_packaging_linux`.
*   **Template:** It uses the `packaging_job.xml.em` template, which is different from the `ci_job.xml.em` template used by other CI jobs.
*   **Build Type:** The `cmake_build_type` is set to `RelWithDebInfo`.
*   **Connext:** `use_connext_debs_default` is set to `true`, which means it will try to install Connext from Debian packages.
*   **Packaging Flag:** The `--packaging` argument is passed to the `run_ros2_batch.py` script, which triggers the packaging logic.
*   **Artifact Archiving:** The job is configured to archive the generated Debian packages (`ws/ros2-package-*.*`).
*   **Docker Image:** The Docker image is tagged as `ros2_packaging`.

## Execution Flow

The execution flow of the `ci_packaging_linux` job is similar to the standard `ci_linux` job, with the addition of the packaging step.

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job uses the standard `linux_docker_resources/Dockerfile`.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile` and tagged as `ros2_packaging`.
    *   **Docker Container Run:** The job runs the newly built Docker image, passing the specific build and test arguments, including the `--packaging` flag, via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py`:** The `ros2_batch_job` Python module within the container handles the build process. When the `--packaging` flag is present, it performs the following additional steps after the build and tests have run:
    *   It generates Debian packages for all the ROS 2 packages in the workspace.
    *   The generated packages are placed in the `ws` directory, ready to be archived by Jenkins.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `ci_packaging_linux` job.
*   **Jenkins Job Template:**
    *   `job_templates/packaging_job.xml.em`: The template for the packaging job.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
*   **Docker configuration:**
    *   `linux_docker_resources/Dockerfile`: The Dockerfile for Linux builds.

## Parameters

The `ci_packaging_linux` job accepts the same parameters as the `ci_launcher` and its other downstream jobs, but some of them are overridden with specific values to configure the packaging build.
