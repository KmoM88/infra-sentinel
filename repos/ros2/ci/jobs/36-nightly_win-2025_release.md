# nightly_win-2025_release Job Analysis

## Overview

The `nightly_win-2025_release` job is a nightly-triggered Jenkins pipeline for building and testing ROS 2 on Windows Server 2025 with release optimizations enabled. It is dynamically generated by the `create_jenkins_job.py` script and uses the `ci_job.xml.em` template.

## Job Generation

The `create_jenkins_job.py` script generates the `nightly_win-2025_release` job for the `windows-2025` OS. It's designed to run a full build and test cycle with a `Release` build type on a nightly basis.

## Release Build-Specific Configuration

The `nightly_win-2025_release` job has the following key configuration settings:

*   **Job Name:** The job is explicitly named `nightly_win-2025_release`.
*   **Template:** It uses the `ci_job.xml.em` template.
*   **Jenkins Node Label:** It is configured to run on a Jenkins node with the `windows-2025-container` label.
*   **Build Type:** The `cmake_build_type` is set to `Release`.
*   **Trigger:** The job is triggered by a timer (`time_trigger_spec`) set to run daily at 4:00 AM (`0 4 * * *`).
*   **Notifications:** Email notifications for build results are sent to `ros2-buildfarm@googlegroups.com`.
*   **Docker Image:** The Docker image is tagged as `ros2_windows_ci_<ROS_DISTRO>`.

## Execution Flow

The execution flow of the `nightly_win-2025_release` job is identical to the `ci_windows-2025` job, with the only difference being the `Release` build type.

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job uses the `windows_docker_resources/Dockerfile`.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile` and tagged as `ros2_windows_ci_<ROS_DISTRO>`.
    *   **Docker Container Run:** The job runs the newly built Docker image, passing the build and test arguments via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py`:** The `ros2_batch_job` Python module within the container handles the build and test process.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `nightly_win-2025_release` job.
*   **Jenkins Job Template:**
    *   `job_templates/ci_job.xml.em`: The template for the CI job.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
*   **Docker configuration:**
    *   `windows_docker_resources/Dockerfile`: The Dockerfile for Windows builds.

## Parameters

The `nightly_win-2025_release` job accepts the same parameters as the `ci_windows-2025` job, but it is typically run with the default values since it is triggered automatically.
