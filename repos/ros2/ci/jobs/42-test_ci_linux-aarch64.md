# test_ci_linux-aarch64 Job Analysis

## Overview

The `test_ci_linux-aarch64` job is a manually-triggered Jenkins pipeline for building and testing ROS 2 on the `aarch64` (ARM64) architecture. It is dynamically generated by the `create_jenkins_job.py` script and uses the `ci_job.xml.em` template. This job is intended for developers to experiment with changes to the CI configuration for the `aarch64` architecture without affecting the main `ci_linux-aarch64` job.

## Job Generation

The `create_jenkins_job.py` script generates the `test_ci_linux-aarch64` job for the `linux-aarch64` OS. It's designed as a sandbox for testing modifications to the CI process for `aarch64`.

## Test Job-Specific Configuration

The `test_ci_linux-aarch64` job has the following key configuration settings:

*   **Job Name:** The job is explicitly named `test_ci_linux-aarch64`.
*   **Template:** It uses the `ci_job.xml.em` template.
*   **Jenkins Node Label:** It is configured to run on a Jenkins node with the `linux_aarch64` label.
*   **Build Type:** The `cmake_build_type` is set to `None`.
*   **Trigger:** This is a manually triggered job and has no time-based trigger.
*   **Notifications:** Email notifications are not configured for this job.
*   **Docker Image:** The Docker image is tagged as `ros2_batch_ci_aarch64`.
*   **Docker Build Argument:** It passes the build argument `PLATFORM=aarch64` to the `docker build` command.
*   **RMW Implementation:** It adds `rmw_connextdds` to the default set of ignored RMW implementations.
*   **Test Arguments:** The `test_args_default` is modified to exclude the `mimick` test.

## Execution Flow

The execution flow of the `test_ci_linux-aarch64` job is identical to the `ci_linux-aarch64` job.

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job uses the standard `linux_docker_resources/Dockerfile`.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile` and tagged as `ros2_batch_ci_aarch64`.
    *   **Docker Container Run:** The job runs the newly built Docker image, passing the build and test arguments via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py`:** The `ros2_batch_job` Python module within the container handles the build and test process.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `test_ci_linux-aarch64` job.
*   **Jenkins Job Template:**
    *   `job_templates/ci_job.xml.em`: The template for the CI job.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
*   **Docker configuration:**
    *   `linux_docker_resources/Dockerfile`: The Dockerfile for Linux builds.

## Parameters

The `test_ci_linux-aarch64` job accepts the same parameters as the `ci_linux-aarch64` job, allowing developers to test a wide range of build and test configurations.
