# nightly_linux_jazzy_coverage Job Analysis

## Overview

The `nightly_linux_jazzy_coverage` job is a nightly-triggered Jenkins pipeline for building and testing the ROS 2 Jazzy Jalisco distribution on Linux and generating code coverage reports. It is dynamically generated by the `create_jenkins_job.py` script and uses the `ci_job.xml.em` template. This job is disabled by default.

## Job Generation

The `create_jenkins_job.py` script generates the `nightly_linux_jazzy_coverage` job for the `linux` OS. It's specifically designed to measure the test coverage of a curated set of packages for the ROS 2 Jazzy release.

## Jazzy Coverage-Specific Configuration

The `nightly_linux_jazzy_coverage` job has the following key configuration settings:

*   **Job Name:** The job is explicitly named `nightly_linux_jazzy_coverage`.
*   **Template:** It uses the `ci_job.xml.em` template.
*   **Jenkins Node Label:** It is configured to run on a Jenkins node with the `linux` label.
*   **ROS Distro:** The `ros_distro` parameter is set to `jazzy`.
*   **Ubuntu Distro:** The `ubuntu_distro` parameter is set to `noble`, which is the target Ubuntu distribution for ROS 2 Jazzy.
*   **Repos URL:** The job uses a specific `.repos` file for the Jazzy distribution: `https://raw.githubusercontent.com/ros2/ros2/jazzy/ros2.repos`.
*   **Build Type:** The `cmake_build_type` is set to `Debug`.
*   **Trigger:** The job is triggered by a timer (`time_trigger_spec`) set to run daily at 4:00 AM (`0 4 * * *`).
*   **Disabled by Default:** This job is disabled by default (`'disabled': True`).
*   **Notifications:** Email notifications for build results are sent to `ros2-buildfarm@googlegroups.com`.
*   **Enable Coverage:** The `enable_coverage_default` parameter is set to `true`.
*   **Build and Test Arguments:** The job uses a specific set of build and test arguments to focus on a curated list of packages (`quality_level_pkgs` and `jazzy_testing_pkgs_for_quality_level`).
*   **Docker Image:** The Docker image is tagged as `ros2_batch_ci`.

## Execution Flow

The execution flow of the `nightly_linux_jazzy_coverage` job is similar to the `nightly_linux_coverage` job, with the main differences being the ROS and Ubuntu distributions used.

1.  **SCM Checkout:** The job checks out the `ros2/ci` repository.

2.  **Docker Environment:** The job runs inside a Docker container.

    *   **Dockerfile Selection:** The job uses the standard `linux_docker_resources/Dockerfile`.
    *   **Docker Image Build:** A new Docker image is built from `Dockerfile` and tagged as `ros2_batch_ci`. The `UBUNTU_DISTRO=noble` build argument is passed to the `docker build` command.
    *   **Docker Container Run:** The job runs the newly built Docker image, passing the specific build and test arguments via the `CI_ARGS` environment variable.

3.  **`run_ros2_batch.py`:** The `ros2_batch_job` Python module within the container handles the build process and generates coverage reports.

## Configuration and Scripts

*   **Job Generation Script:**
    *   `create_jenkins_job.py`: The script that generates the `nightly_linux_jazzy_coverage` job.
*   **Jenkins Job Template:**
    *   `job_templates/ci_job.xml.em`: The template for the CI job.
*   **Build Logic:**
    *   `run_ros2_batch.py`: The entry point for the build process inside the Docker container.
*   **Docker configuration:**
    *   `linux_docker_resources/Dockerfile`: The Dockerfile for Linux builds.

## Parameters

The `nightly_linux_jazzy_coverage` job accepts the same parameters as the `ci_linux_coverage` job, but many of them are overridden with specific values to configure the Jazzy coverage build.
